⚡ Proxi New — ZombieCoder Bridge Server

সম্পূর্ণ প্রজেক্ট ডকুমেন্টেশন • কোনো কোড ছাড়া কিভাবে কাজ করে • নির্ভরতা ও সংযোগ

📋 সারসংক্ষেপ

Proxi New হলো একটি Local-first AI execution engine — একটি সাধারण Express.js সার্ভার যেটি বিভিন্ন LLM প্রোভাইডারকে একটি OpenAI-compatible API-তে একত্রিত করে। এটি:

  • মাল্টিপল LLM প্রোভাইডার (OpenCode Zen, Groq, Ollama, OpenAI) কে একটি সিঙ্গেল API endpoint-তে জোড়া দেয়
  • স্মার্ট রাউটিং দিয়ে সেরা মডেল অটো-সিলেক্ট করে
  • RAG (Retrieval-Augmented Generation) সিস্টেম দিয়ে প্রজেক্ট ডকুমেন্টেশন বুঝে
  • MCP (Model Context Protocol) দিয়ে এডিটর সংযুক্ত করে
  • বিস্তারিত Admin Dashboard দিয়ে সব মনিটর ও ম্যানেজ করে
  • SQLite ডেটাবেসে সব অবস্থা সংরক্ষণ করে

১. সার্ভার চালু হওয়ার ধাপ

ধাপ ১: .env ফাইল থেকে সব কনফিগ পড়ে (API keys, ports, models)
→
ধাপ ২: state.db (SQLite) তৈরি করে — সব tables create হয়
→
ধাপ ৩: ৬টি LLM প্রোভাইডার থেকে মডেল লোড করে (sync)
→
ধাপ ৪: Agent system চালু হয় (RAG, SSOT, Workspace Watcher)
→
ধাপ ৫: Port 9999-তে লিসেন করা শুরু করে

২. একটি Chat Request কিভাবে যায়

Client: POST /v1/chat/completions পাঠায়
→
Auth Middleware: API key যাচাই করে
→
OpenAI Controller: Identity prompt যোগ করে system message-এ
→
MawlanaRouter: Input analysis করে → সেরা মডেল সিলেক্ট করে
→
GroqService: নির্বাচিত LLM provider-কে request পাঠায়
→
Rate Limiter: মডেল-ভিত্তিক rate limit চেক করে
→
Response: OpenAI format-এ client-কে ফেরত দেয়
→
DB Logger: usage_stats-এ log লেখে

৩. Agent System কিভাবে কাজ করে

Editor (VS Code): MCP protocol দিয়ে connect হয়
→
MCP Controller: JSON-RPC request handle করে
→
Agent Controller: directory auto-trust করে
→
DiskRAGService: প্রজেক্ট scan করে → SSOT.md তৈরি/আপডেট করে
→
Workspace Watcher: ফাইল পরিবর্তন হলে SSOT auto-refresh
→
VectorIndexService: embedding তৈরি করে → search-এ ব্যবহার হয়

📐 মডেল রাউটিং — কিভাবে সিদ্ধান্ত হয়

MawlanaRouter — স্মার্ট মডেল সিলেক্টর

প্রতিটি request-এর জন্য MawlanaRouter নিম্নলিখিতভাবে সেরা মডেল বাছাই করে:

  1. Input analysis: message content পড়ে — কোডিং? চ্যাট? RAG?
  2. Category detection: chat, code, rag, guard — একটি নির্বাচন
  3. Preference list: প্রতিটি category-র জন্য একটি ordered model list আছে
  4. Rate limit check: মডেলটি কি available? Rate limit পার হয়নি?
  5. Fallback: প্রথম choice কাজ না করলে পরবর্তী model try করে
Default Model: mimo-v2.5-free (OpenCode Zen — ফ্রি, কোনো API key লাগে না)
Fallback: llama-3.3-70b-versatile (Groq — ফ্রি tier)

🛡️ নিরাপত্তা ও নিয়মাবলী

🔑 Authentication

  • GROQ_API_KEY → Groq API-তে যাওয়ার জন্য
  • OPENCODE_API_KEY → OpenCode Zen-এর জন্য
  • ADMIN_SECRET → Admin dashboard protection (ঐচ্ছিক)
  • API key না থাকলে: 500 error দেয়
  • ফাঁকা Bearer token: 401 error দেয়
  • Admin secret না থাকলে: dev mode (open access)

🛡️ SQL Injection Protection

  • সব table name whitelist-এর মধ্যে সীমাবদ্ধ
  • 12টি whitelisted table: identity, models, sessions ইত্যাদি
  • অন্য কোনো table name দিলে: error
  • ব্যক্তিগত তথ্য (identity.json) .env-তে সরানো হয়েছে

🔒 XSS Protection

  • সব user input escapeHtml() দিয়ে sanitize
  • Dashboard HTML-তে model names, identity values
  • API responses: Content-Type headers

📋 Rate Limiting

  • প্রতিটি model-এর জন্য আলাদা RPM/TPM limit
  • Default: 30 RPM, 60000 TPM
  • Rate limit পার হলে: auto-switch fallback model
  • 5-minute window tracking
